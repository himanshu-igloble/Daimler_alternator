## Fleet Feature Contribution Decomposition — LOVO out-of-fold ordering

**File:** `visualizations/contribution/fleet_decomposition_LOVO.png`
**Companion (full-fit ordering):** `visualizations/contribution/fleet_decomposition.png`

### What the chart is
An **additive decomposition of the Ridge classifier's linear decision score**, one stacked bar per truck. The model scores each truck as:

```
risk_linear = intercept + Σₖ (coefₖ × zₖ)
```

- **zₖ** — the truck's **standardised** value for feature *k*: `z = (x − fleet_mean) / fleet_std` (z>0 = above fleet average; z<0 = below).
- **coefₖ** — the **fixed, learned** Ridge weight for feature *k* (identical across all trucks).
- **Each coloured segment = coefₖ × zₖ** — that feature's signed contribution, in score units, for that truck.
- **Net bar height ≈ risk_linear** (the six segments summed, plus a constant intercept ≈ −0.20).

Intercept ≈ −0.20 (the ±1-class-balance offset: 10 failed / 15 healthy).

### How to read it (5 moves)
1. **One bar = one truck.** Name colour: **red = actually failed, green = healthy.**
2. **The black line at 0 is the divide.** Segments **above** push toward *fail*; **below** push toward *healthy*. The **net height** is the truck's risk score.
3. **Each colour is one signal's vote**, sized by `coef × z`. Big segment ⇒ the truck is an outlier on that signal; direction ⇒ toward fail (up) or healthy (down). The **same colour can point up for one truck and down for another** (sign = sign(coef) × sign(z)).
4. **Left → right = the honest LOVO ranking.** Each truck is ordered by its **Leave-One-VIN-Out out-of-fold score** (scored by a model that never trained on it) — the deployment-grade ranking behind **AUROC 0.9267**. Left = lowest predicted risk, right = highest.
5. **Read the verdict** from position + height: healthy bars cluster low/left, failed bars cluster high/right; net (not gross) height is the model's call.

### Why the LOVO ordering (vs the full-fit copy)
The contributions are computed from the in-sample full-fit inspection model; this chart only **re-orders** them by the honest LOVO score. The two orderings are **Spearman 0.988** correlated. Ordering by the in-sample score implies AUROC **0.947**; the honest LOVO order is **0.927** — the ~2-point gap is in-sample optimism, not a different result. **14/25 trucks shift by ≤2 positions; none cross from clearly-risky to clearly-healthy.** The visible honesty markers:
- **VIN5_F_ALT** (false negative, LOVO prob 0.28) sits **among the healthy cluster on the left** — the missed failure.
- **VIN3_NF_ALT** (false positive, 0.49) sits **inside the failed cluster on the right** — the one false alarm.

So this chart shows the true, slightly-messier separation the **deployable** model achieves, not the marginally cleaner in-sample picture.

### What it demonstrates
- **No single signal decides it.** The strongest feature alone separates failed/healthy at only AUROC ≈ 0.69; the score is the *sum* of six modest pushes — the stacking is what lifts the ensemble to 0.927.
- **Ordering = the AUROC.** `risk_linear` maps monotonically to probability via a sigmoid, so this left-to-right order *is* the ranking that produces the AUROC; the Youden operating threshold (prob 0.4456) is a vertical cut in the right third.
- ⚪ **progressive_drift points "the wrong way"** by design (negative coef): an exposure artifact (longer-lived healthy trucks accumulate more drift); lowest importance (0.048), retained only to preserve the frozen V10.5.3 spec.
