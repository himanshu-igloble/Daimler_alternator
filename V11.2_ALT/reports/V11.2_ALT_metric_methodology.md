# V11.2_ALT — How the Coefficient & Importance Metrics Are Calculated

**Date:** 2026-06-24  |  **Scope:** definitions, formulas, and sources for every per-feature
number shown in the contribution and importance charts.

The risk-ranking model is a frozen **L2-regularised linear classifier (ridge regression on
±1 labels), α = 1.0**, fit on six standardised features. Four different per-feature numbers
are reported; they answer four different questions and are computed independently. This doc
explains each.

---

## 0. Preprocessing — standardisation (shared by all)

Every feature is **z-scored** before the model sees it:

```
zᵢₖ = (xᵢₖ − μₖ) / σₖ
```

- `xᵢₖ` = raw value of feature *k* for truck *i*; `μₖ, σₖ` = fleet mean / std of feature *k*.
- After this, `z > 0` means "above fleet average", `z < 0` means "below".
- Standardisation makes coefficients **directly comparable** (each is the effect of a 1-standard-deviation move), and is why the importance bars can be read side by side.

Source: `StandardScaler` in `V11.2_ALT/src/V11_2_ALT_weightage_contribution.py`.

---

## 1. Coefficient  (`coefₖ`)  — the model's learned weight

**What it answers:** in which direction, and how strongly per 1 SD, does each signal move the risk score?

**How it is computed.** The linear model solves the L2-penalised least-squares problem on
±1-encoded labels:

```
β = argmin ‖ Z·β − y ‖²  +  α·‖β‖²        (α = 1.0)
  = (ZᵀZ + αI)⁻¹ Zᵀ y
```

- `Z` = standardised feature matrix (25 × 6), `y ∈ {−1, +1}` (failed = +1).
- `coefₖ` = the *k*-th entry of `β`. **Sign** = direction (positive ⇒ higher value raises failure risk); **magnitude** = strength per 1 SD.
- The **intercept** `β₀ ≈ −0.20` is the class-balance offset: `(n_fail − n_healthy)/n = (10 − 15)/25 = −0.20`.
- This is **inspection** of the frozen V10.5.3 model — we refit with the identical spec only to read `β`; we never retrain for deployment.

**Per-truck linear score** is the additive sum:

```
risk_linearᵢ = β₀ + Σₖ coefₖ · zᵢₖ
```

Source: `V11_2_ALT_weightage_contribution.py` → `results/V11.2_ALT_weightage_summary.json`
(`ridge_coefficients`).

---

## 2. |Mean contribution|  — in-sample statistical weight

**What it answers:** on average, how much does this signal actually *move* the scores of the trucks in this fleet?

**How it is computed.** Each feature's signed contribution to one truck's score is `coefₖ · zᵢₖ`.
The per-feature weight is the mean of its **absolute** contribution across all 25 trucks:

```
|mean contribution|ₖ = (1/N) Σᵢ | coefₖ · zᵢₖ |  =  |coefₖ| · mean_i(|zᵢₖ|)
```

- It blends the coefficient (`|coefₖ|`) with **how spread-out the fleet is** on that feature (`mean|z|`).
- **Outlier-sensitive:** a few trucks with extreme `z` inflate it. A feature can score high here yet not help prediction (see §5).

Source: `V11_2_ALT_weightage_contribution.py` → `results/V11.2_ALT_contribution.csv`
(grouped) and `..._weightage_summary.json` (`per_feature_mean_abs_contribution`).

---

## 3. Permutation importance  — generalisation reliance

**What it answers:** how much does the model's *ranking quality* depend on this signal (out of sample)?

**How it is computed.** With the fitted model held fixed:

```
for r = 1 … R (R = 100):
    randomly shuffle column k across trucks (break its link to the outcome)
    AUROC_perm[r] = AUROC of the model with feature k shuffled
importanceₖ = AUROC_full − mean_r( AUROC_perm[r] )
```

- The bigger the AUROC drop when a feature is scrambled, the more the model **relies** on it.
- It is **model-aware** and accounts for redundancy: if a feature's information is duplicated by
  others, shuffling it barely hurts AUROC → low importance, even if its coefficient is large.
- Computed during the V10.5.3 build over the evaluation folds (`permutation_n = 100`).

Source: `V5.2_ALT/models/classification/V10.5.3_20_5_ALT_ridge_spec.json` (`feature_importance`),
mirrored in `..._weightage_summary.json` (`perm_importance`).

---

## 4. Univariate AUROC  — standalone separation

**What it answers:** using this one signal *alone*, how well are failed and healthy trucks separated?

**How it is computed.** Rank all trucks by the single feature and form every (failed, healthy) pair
(10 × 15 = 150 pairs):

```
univariate AUROCₖ = ( #concordant + 0.5·#ties ) / (n_fail · n_healthy)
```

- This equals the normalised **Mann–Whitney U** statistic.
- **0.5 = no separation**; **> 0.5** = higher in failed trucks; **< 0.5 = inverted** (higher in healthy).
  `progressive_drift` = 0.26 ⇒ inverted.
- Ignores all other features (no model involved).

Source: `V11.2_ALT/src/V11_2_ALT_heuristic_stats.py` → `results/V11.2_ALT_heuristic_stats.csv`
(`auroc`), using `auroc_from_scores` in `V11_2_ALT_common.py`.

---

## 5. Why the three importance views disagree (and that's expected)

| View | Measures | Sensitive to |
|---|---|---|
| \|mean contribution\| | in-sample score movement | a few outlier trucks |
| permutation importance | out-of-sample ranking reliance | redundancy between features |
| univariate AUROC | standalone separation | nothing else (single feature) |

**Worked example — `bat_charge_delta_trend_right` (the clearest divergence):**
- coefficient `+0.224`, and `mean|z| = 0.170 / 0.224 ≈ 0.76` — a spread similar to the top feature, so it **swings scores** (|mean contribution| = 0.170, rank #3).
- yet permutation importance is only **0.034 (rank #6, last)** — shuffling it barely changes the ranking, because its information is largely **redundant** with the other voltage signals.
- and standalone it is near-random: **univariate AUROC 0.56 (rank #5)**.
- → it looks influential in-sample but **does not generalise**; a few outlier trucks drive its contribution.

**Worked example — `vsi_std_ratio_30d` (top model feature):**
- coefficient `+0.443`, |mean contribution| `0.348` (#1), permutation importance `0.155` (#1) — the model relies on it most.
- but standalone univariate AUROC is only **0.69 (#3)** — alone it is a modest separator. Its power comes from the **combination** with the other five signals, which is why the six-feature ensemble reaches AUROC 0.927 when no single feature exceeds ~0.81.

**Bottom line:** coefficient/contribution describe *this* fleet; permutation importance describes
*generalisation*; univariate AUROC describes *each signal on its own*. Read together they reveal
which signals are genuinely predictive (high on all three), which are redundant (high contribution,
low importance), and which are artifacts (inverted univariate AUROC).
