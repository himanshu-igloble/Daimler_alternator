# Can a per-truck USAGE clock beat the fleet CALENDAR clock?

**Question.** Section 1's RUL is fleet-based (a function of calendar age only). A usage clock models km-to-failure and converts to days via each truck's km/day, so a hard-working truck burns RUL faster. Does it predict remaining life better?

## Method (honest)

- Leave-one-VIN-out: score each failure with a Weibull fit on the **other 9** only.
- Three independent inputs: `TTF_days = JCOPENDATE - t0`, `KTF_km = est_km`, `kpd = km_per_day_est`.
- Conditional median remaining at install (f=0), mid-life (f=0.5), late (f=0.8); MAE vs actual `(1-f)*TTF`.

## Deciding diagnostic

| spread across the 10 failures | CV |
|---|---|
| days-to-failure (calendar) | **0.103** |
| km-to-failure (usage) | **0.169** |


The usage clock can only win if km-to-failure is the tighter quantity. Here **days is tighter -> calendar should win**.

## Result — LOVO conditional-median RUL MAE (days)

| horizon | calendar MAE | usage MAE | winner |
|---|---|---|---|
| install (predict full life) | 52.2 | 95.6 | **calendar** |
| mid-life (50%) | 52.2 | 91.2 | **calendar** |
| late (80%) | 49.8 | 82.8 | **calendar** |
| **overall** | **51.4** | **89.9** | **calendar** |

Historical fleet calendar-clock baseline ≈ 50 d; the per-truck *feature* (beta) attempt ≈ 142 d.

## Verdict

**CALENDAR clock WINS overall (90 vs 51 d MAE).**

The usage clock does **not** beat the calendar fleet clock at n=10 failures: alternators here fail at a more consistent **age** than mileage, so personalizing the clock by usage adds error. This re-confirms the standing finding — at this data scale the RUL **number** stays the fleet window (601 d / ~120k km); per-truck intelligence lives in Section 1b (condition), Section 0 (effective alert) and the GED emergency channel, not in a per-truck RUL number. A usage clock becomes worth revisiting only with many more failure events (the 500-truck scale-up).
