## 1. GED Per-State Physical Interpretation

GED (project name for the CAN signal "State of Alternator Excitation") is defined in the KT as a 2-bit discrete enum. Below is the cross-validated interpretation of each state.

| GED Value | KT Name | Physical Interpretation | External Evidence | Confidence |
|-----------|---------|------------------------|-------------------|------------|
| **0** | No disturbance | Alternator excitation circuit is operating normally; voltage regulator is actively exciting the rotor field; charging output expected at engine-running conditions. | Consistent with J1939 2-bit convention where 00b = "normal/no-fault" state. The dominant operating state for any healthy fleet (>99.7% of rows in KT fleet data) aligns with expectations. | **High** |
| **1** | Not allowed | The ECU has inhibited/suppressed alternator excitation — a commanded state where the powertrain controller deliberately prevents charging (e.g., during aggressive load-shedding, cold-start strategies, or regenerative-braking torque management). | J1939 2-bit convention where 01b = "enabled/inhibited/second valid state" (value meaning varies by parameter). ECU-commanded excitation inhibit is a documented real function in smart charging systems where the ECU can suppress excitation via the IG (ignition-enable) or LIN "VSET=0" command. Bosch CR665 and NXP AR6000 support programmatic excitation suppression via LIN. Never observed in the 59-truck DICV fleet — consistent with a rarely-commanded mode. | **Medium** (functionally plausible; never observed in this fleet so empirical confidence is N/A) |
| **2** | Disturbance | An active excitation fault has been detected: the excitation circuit has deviated from normal in a way the ECU or alternator regulator has classified as a fault condition. Most likely causes: brush/rotor winding degradation, rectifier fault, voltage regulator instability, or open D+/WL circuit. | Consistent with J1939 2-bit convention where 10b = "error indicator." The 485x enrichment in failed-alternator trucks (0.31% vs 0.0006%) is strong empirical confirmation that GED=2 encodes a real fault condition. External OEM fault architectures (Daimler DTC table in KT: entries 66F7E4, 67F7E4 for D+/WL faults) confirm this kind of fault is flagged discretely. | **High** |
| **3** | Signal not available | The CAN frame carrying GED was lost, late, or the ECU source had no valid data at sample time (CAN bus comms fault, sensor-node timeout). Not a physical alternator fault — a telemetry gap. | Consistent with J1939 2-bit convention where 11b = "not available / signal not supported." The KT data confirms GED=3 appears in both failed and healthy trucks (per-VIN rates 0–29%), confirming it is a communication artefact, not a component-health indicator. This is the F-2 FAIL finding in domain_data_verification.md (GED=3 is NOT failure-exclusive). | **High** |

> **Key physical chain:** The GED signal is generated by the ECU (CPC or equivalent) from its monitoring of the alternator's WL (Warning Lamp) pin, D+ (excitation circuit positive) pin, and/or LIN regulator fault flags. The ECU summarises these hardware-level signals into a 4-state CAN broadcast. This is consistent with the KT architecture: new-generation vehicles use an ASAM module to monitor WL/D+ and publish status to the vehicle CAN bus.

## 2. J1939 2-Bit Status Analysis + Candidate SPN

### 2.1 The J1939 2-Bit Convention

SAE J1939-71 formally defines a 2-bit discrete parameter convention where:
- **00b (0)** = valid state A (typically "normal", "no fault", "disabled")
- **01b (1)** = valid state B (typically "enabled", "active", "inhibited")
- **10b (2)** = error indicator
- **11b (3)** = not available / signal not supported

**Source:** CSS Electronics, "J1939 Explained" (https://www.csselectronics.com/pages/j1939-explained-simple-intro-tutorial): "For 2-bit discrete parameter signals, J1939-71 states that the value 2 reflects an error and the value 3 reflects not available."

**Important caveat (same source):** "In our experience this seems to contradict the J1939 Digital Annex SPN descriptions, where several 2-bit signals utilise the full set of 4 potential values to communicate valid data." This means the convention is a default, not a universal rule — some 2-bit SPNs use values 0–3 all as valid data states.

**Confidence:** High that the convention exists; Medium that GED specifically follows it (it is consistent, but no authoritative source confirms this for GED's exact implementation).

### 2.2 GED Mapping vs J1939 2-Bit Convention

| GED | GED Name | J1939 2-Bit Canonical | Match Quality |
|-----|----------|-----------------------|---------------|
| 0 | No disturbance | 00b = normal/no fault | **Exact match** |
| 1 | Not allowed | 01b = second valid state (enabled/inhibited) | **Plausible match** — "inhibit" = ECU actively suppressing excitation |
| 2 | Disturbance | 10b = error indicator | **Exact match** — "disturbance" = error/fault |
| 3 | Signal not available | 11b = not available | **Exact match** |

The GED encoding is **structurally identical** to the J1939 2-bit convention. Values 0, 2, and 3 map unambiguously; value 1 ("Not allowed") aligns with the ECU-commanded inhibit function.

### 2.3 Candidate SPN/PGN in J1939

| J1939 Parameter | PGN | Notes | Relevance |
|----------------|-----|-------|-----------|
| **Voltage Regulator Excitation Status (VREP)** | **64934 (0x00FDA6)** | Defined in J1939-75 (power generation). Name matches very closely. | High conceptual match; but J1939-75 is for stationary/marine generators, not vehicle alternators. Cannot confirm this is the SPN behind GED without access to proprietary Daimler DBC. |
| **Alternator 1–4 Status** | Not confirmed | Listed in HEM Data J1939 parameter database as discrete status parameters (https://hemdata.com/support/list-of-heavy-duty-j1939-parameters/). Exact SPN numbers and bit definitions are in the proprietary J1939 Digital Annex. | Moderate relevance — discrete "Alternator N Status" parameters do exist in J1939. |
| **VEP1 Vehicle Electrical Power (PGN 65271)** | 65271 | Contains SPN 168 (battery voltage, continuous), SPN 167 (charging system voltage, continuous), SPN 115 (alternator current, continuous). No discrete excitation status field. | Low — these are continuous measurements, not enums. |

### 2.4 Adversarial Self-Check: Alternatives to J1939

Three equally plausible alternatives to the J1939 mapping were evaluated:

1. **Daimler/OM471 proprietary CAN SPN (range 516096–524287):** Proprietary SPNs in J1939 are OEM-specific and not publicly documented. GED could be a Daimler-proprietary 2-bit CAN signal (published only in the Daimler DBC / XENTRY system). This is the **most likely concrete implementation** — the encoding pattern mirrors J1939 convention but the specific SPN is proprietary. Confidence: Medium.

2. **ISO 11992 (truck/trailer bus) status encoding:** European heavy trucks frequently use ISO 11992-2 for truck-trailer electrical communication, which has its own discrete fault encoding. The 4-state pattern is also consistent with ISO 11992 multi-state fault indicators. Not confirmed because the DICV fleet uses a different communication path. Confidence: Low.

3. **Bosch-proprietary LIN-to-CAN bridge status byte:** Bosch alternator regulators (CR665, etc.) communicate via LIN to the ECU; the ECU translates fault flags into a CAN byte. Bosch-specific status encoding could yield a 4-state result. Plausible but not confirmed from public Bosch documentation. Confidence: Low.

### 2.5 J1939 Verdict

> **PLAUSIBLE, UNCONFIRMED.** The GED 4-state encoding is structurally consistent with the SAE J1939 2-bit status convention, and a J1939-75 PGN named "Voltage Regulator Excitation Status" (64934) is a conceptually strong match. However, no publicly accessible source directly confirms that GED maps to a specific J1939 SPN in J1939-71 (vehicle application). The most defensible conclusion is that GED follows the **J1939 2-bit convention pattern** — possibly because Daimler's proprietary implementation was designed to be J1939-compatible — but the specific SPN is a Daimler-proprietary value not documented in open sources.

## 3. OEM Enum vs Duty-Cycle Evidence

### 3.1 The Two-Layer Architecture

Modern smart alternator systems have **two distinct communication layers**:

| Layer | What is transmitted | Format | Where |
|-------|--------------------|---------|----|
| **LIN bus (alternator ↔ ECU)** | Excitation PWM duty cycle (continuous), excitation current (continuous), temperature (continuous), fault flags (discrete bits) | Continuous + individual fault bits | Internal, not visible in fleet telematics |
| **Vehicle CAN bus (ECU → telematics)** | Processed status enum summarising excitation health | Discrete enum (2–4 states) | Captured by fleet telematics as GED |

### 3.2 OEM Evidence Table

| OEM / Supplier | Signal Type Reported | Medium | Notes | Source | Confidence |
|----------------|---------------------|--------|-------|--------|------------|
| **Bosch (CR665)** | LIN2.1 interface; reports excitation current and fault categories (mechanical, electrical, temperature, communication, timeout) to ECU | LIN continuous + fault bits | ECU then publishes discrete status on CAN | https://www.bosch-semiconductors.com/products/automotive-ics/ice-powertrain/cr665/ | High |
| **NXP / Freescale (AR6000)** | Excitation duty cycle (continuous), excitation current (continuous) via LIN | LIN continuous | Fault categories separately flagged | https://www.nxp.com/docs/en/data-sheet/AR6000.pdf (URL; PDF not accessible) | Medium |
| **Infineon (TLE8881-2)** | RDC register: duty cycle (5-bit, 0–100%), RMC register: excitation current (0.05A resolution, 8-bit continuous), RMT: temperature | LIN continuous registers | No single 4-state enum observed in public datasheet; individual fault bits reported separately | https://www.alldatasheet.net/html-marking/1238755/INFINEON/TLE8881-2-CH/30287/66/TLE8881-2-CH.html | High (from readable page) |
| **STMicroelectronics (L9918)** | Duty cycle continuous, LIN bus v1.3/2.1/2.2 | LIN continuous + state machine states | State machine has pre-excitation, regulation, default, and wake states — discrete at the operational level but these are internal regulator modes | https://www.st.com/resource/en/datasheet/l9918.pdf | Medium (PDF timed out; cited from search snippets) |
| **Mercedes-Benz (GenCOM/GenMON)** | GenCOM (ECU→alternator): PWM sets target voltage; GenMON (alternator→ECU): PWM reports excitation duty cycle | Continuous PWM (bilateral) | Used in 12V passenger cars. 24V truck may use LIN instead. The ECU processes GenMON and publishes a discrete status on CAN. | https://www.benzworld.org/threads/alternator-really-dead-or-no-pwm-signal.3056372/ | Medium |
| **Daimler truck (CPC/ASAM architecture)** | WL and D+ pin monitoring → ECU creates 4-state enum on CAN | Discrete enum | Described in KT: WL = warning lamp (open = fault), D+ = excitation indicator (open circuit triggers WL). ASAM module monitors these and publishes to CAN. | KT_daimler/KT_startermotor_alternator.md (internal) | High |

### 3.3 Key Finding: Enum vs Duty Cycle at Each Layer

- **At the LIN/regulator layer:** Excitation is reported as a **continuous duty cycle** (percentage) and a **continuous current** measurement. These are raw physical measurements.
- **At the vehicle CAN bus layer:** The ECU translates regulator fault flags + hardware pin states (WL/D+) into a **discrete status enum** — which is what fleet telematics captures as GED.
- The "duty-cycle" hypothesis for GED is inconsistent with GED taking only 4 discrete integer values {0, 1, 2, 3} in the fleet data. A duty-cycle signal would have continuous resolution.

## 4. LIN Smart-Alternator Excitation Reporting

LIN alternator regulators (the dominant technology for smart alternators in commercial vehicles from ~2010 onward) expose excitation state through two mechanisms:

### 4.1 Continuous Duty Cycle (Primary)

The excitation PWM duty cycle is the primary control variable. It is measured and reported back via LIN as a continuous register:

- **Infineon TLE8881-2:** Register `RDC[4:0]` = excitation duty cycle, 5-bit resolution (32 steps, ~3.1% per step). Formula: DC = n × 100%/32.
  Source: https://www.alldatasheet.net/html-marking/1238755/INFINEON/TLE8881-2-CH/30287/66/TLE8881-2-CH.html
- **Infineon TLE8881-2:** Register `RMC[7:0]` = measured excitation current, 8-bit, resolution 0.05A/bit or 0.04A/bit (variant-dependent). Continuous measurement.
- **Electronic Design (LIN alternator article):** "Signals transmitted to the ECU include field excitation duty cycle, field current or temperature (customer-defined)."
  Source: https://www.electronicdesign.com/news/article/21187534/lin-controlled-alternator-voltage-regulator
- **WAI Global COM Regulators bulletin** confirms LIN regulators transmit "field excitation duty cycle, field current, VSP, temperature, and fault status."
  Source: https://www.waiglobal.com/media/wysiwyg/1177-01-WAI-US-CR-EN%20COM%20Regulators.pdf

### 4.2 Fault Status (Discrete Flags)

LIN regulators also flag distinct fault categories as discrete bits:

- Mechanical fault flag
- Electrical fault flag
- Temperature fault flag
- Communication/timeout fault flag

These are **individual bits**, not a unified 4-state enum. The ECU that receives these bits is responsible for synthesising them into a single status value for the CAN bus (such as GED).

### 4.3 LIN Node Profile

The LIN alternator node profile (specified in LIN Specification Package 2.1 by LIN-CiA) defines the communication protocol between alternator and ECU. The node profile specifies:
- Frame identifiers for command (ECU→alternator) and response (alternator→ECU)
- Response fields: duty cycle, current, temperature (all continuous)
- Fault/status flags: individual bits per fault category

Source: https://www.lin-cia.org/fileadmin/microsites/lin-cia.org/resources/documents/LIN-Spec_Pac2_1.pdf

> **Conclusion for Q3:** In LIN smart alternators, excitation is reported as a **continuous duty cycle** to the ECU, not as a status code. The 4-state discrete status visible on the vehicle CAN bus (GED) is generated by the ECU as a processed summary, not read directly from the LIN node.

> **Key finding:** In standard J1939 fleet telematics (VEP1), the alternator-related signals that are routinely captured are **voltage** (continuous) and potentially **current** (continuous). The **excitation field current** (raw LIN measurement from regulator RMC register) is NOT typically exposed on the vehicle CAN bus in fleet telematics feeds — it remains on the LIN bus between alternator and ECU. What IS exposed is the ECU-synthesised discrete status (like GED), which summarises fault conditions without quantifying the continuous duty cycle or current value.

## 7. Bottom Line: Status Enum or Quantized Duty Cycle?

### Verdict: **DEFINITIVELY A STATUS ENUM. High confidence.**

| Question | Answer | Confidence |
|----------|--------|------------|
| Is GED a discrete status enum? | **Yes.** Only 4 integer values observed {0,1,2,3}. Continuous duty cycle would show fractional values across the full range. | **High** |
| Is GED a quantized duty cycle? | **No.** Refuted by discrete data + two-layer architecture: duty cycle lives on LIN, enum lives on CAN. | **High** |
| Does GED follow J1939 2-bit status convention? | **Plausible, unconfirmed.** Structural match is exact for all 4 values. J1939-75 PGN 64934 has the right name. No public source confirms the specific SPN for a Daimler vehicle alternator. | **Medium** |
| Is there an alternative to J1939? | **Yes — Daimler-proprietary SPN (range 516096–524287).** The encoding pattern mirrors J1939 but the specific SPN is not in any public database. This is the most likely concrete implementation. | **Medium** |
| Does any OEM expose field/excitation current continuously on the vehicle CAN bus? | **Rarely in fleet telematics.** J1939 VEP1 includes alternator current (SPN 115) but field excitation current is typically kept on the LIN bus. | **Medium** |

### Physical interpretation summary

GED is generated by the CPC/ECU from its monitoring of the alternator excitation circuit (D+ and WL hardware pins, and/or LIN fault flags from the voltage regulator). The ECU synthesises a 4-state enum that encodes:
- State 0: excitation circuit operating normally
- State 1: ECU has commanded excitation off (load-shedding or protection)
- State 2: excitation fault detected (rectifier, brush, winding, or regulator anomaly)
- State 3: the CAN frame or sensor source was unavailable at sample time

This interpretation is internally consistent with the KT, consistent with J1939 2-bit convention, and consistent with Daimler's known alternator monitoring architecture (WL/D+ → ASAM/CPC → CAN bus).
